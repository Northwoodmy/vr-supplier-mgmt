# 用户权限说明文档

## 概述

本系统采用基于角色（RBAC）的权限管理体系，用户通过分配角色来获得相应的权限。系统支持多角色分配，用户拥有其所有角色的权限并集。

## 用户类型与角色

系统预定义了三种角色，每种角色对应不同的权限级别：

| 角色 | 标识名 | 说明 |
|------|--------|------|
| 超级管理员 | `super_admin` | 拥有系统全部权限，包括用户管理、审计日志查看等高级功能 |
| 管理员 | `admin` | 管理供应商、项目和评估业务，可查看所有报表和产能数据 |
| 普通用户 | `user` | 基础业务操作权限，可创建和编辑供应商、项目、评估记录 |

## 权限详细说明

### 权限码说明

权限码采用 `模块：操作` 的命名规范：
- 第一部分：所属业务模块（如 `user`、`supplier`、`project`）
- 第二部分：允许的操作类型（如 `manage`、`create`、`edit`、`delete`、`view`）

## 权限检查机制

### 前端权限控制

```typescript
// 示例：在组件中检查权限
const { data: user } = useAuth();
const canDelete = user?.permissions?.includes('supplier:delete');

// 条件渲染
{canDelete && (
  <Button onClick={handleDelete}>删除</Button>
)}
```

### 后端权限保护

```typescript
// 示例：tRPC 路由权限检查
export const deleteSupplier = protectedProcedure
  .input(z.object({ id: z.string() }))
  .mutation(async ({ ctx, input }) => {
    if (!ctx.user?.permissions?.includes('supplier:delete')) {
      throw new TRPCError({
        code: 'FORBIDDEN',
        message: '无权执行此操作',
      });
    }
    // 删除逻辑...
  });
```

## 侧边栏导航权限

系统根据用户权限动态显示导航菜单：

- **主导航**：所有登录用户可见
  - 仪表盘、供应商管理、项目管理、产能评估、品质评估、报表统计

- **系统管理**（仅对有权限用户显示）
  - 用户管理：需要 `user:manage` 权限
  - 审计日志：需要 `audit:view` 权限

## 权限扩展

如需添加新权限，请按以下步骤操作：

1. **定义新权限码**：在 `src/lib/db.ts` 的角色配置中添加
2. **后端保护**：在 API 路由中添加权限检查
3. **前端控制**：在 UI 组件中根据权限条件渲染

示例：
```typescript
// 添加新权限到角色
permissions: [
  // ...现有权限
  'new_module:operate',
]

// 后端检查
if (!ctx.user?.permissions?.includes('new_module:operate')) {
  throw new TRPCError({ code: 'FORBIDDEN' });
}
```

## 安全建议

1. **最小权限原则**：用户只应被授予完成工作所需的最小权限
2. **定期审计**：通过审计日志定期检查权限使用情况
3. **角色审查**：定期审查用户角色分配是否合理
4. **密码策略**：强制执行强密码策略，定期更换密码
5. **会话管理**：敏感操作需要重新验证身份
