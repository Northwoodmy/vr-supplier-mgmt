## 二、发现的问题

### 🔴 严重问题（阻碍功能使用）

#### 1. 登录功能无法正常工作 - 【严重】

**问题描述**: 
用户点击登录按钮后，虽然 API 返回成功，但前端没有跳转到仪表盘，而是停留在登录页。

**复现步骤**:
1. 访问 http://10.10.10.39:23456/login
2. 输入用户名 admin 和密码 Admin@123
3. 点击登录按钮
4. 页面不跳转，仍停留在登录页

**原因分析**:
`src/app/api/auth/login/route.ts` 中使用了 `cookies().set()` 设置 session cookie，但在 Next.js App Router 的 Route Handler 中，这样设置的 cookie 不会发送到客户端。必须通过 `NextResponse` 的 `headers` 来设置 `Set-Cookie`。

**影响**: 
用户无法登录，整个系统无法使用。

**相关文件**:
- `/src/app/api/auth/login/route.ts:74-77`
- `/src/components/auth/login-form.tsx`

**问题描述**:
项目计划中定义的供应商管理、项目管理、产能评估、品质评估、报表统计等核心功能的 tRPC 路由器文件**不存在**。

**缺失文件**:

- `src/server/trpc/routers/evaluation.ts` - 品质评估 API
- `src/server/trpc/routers/report.ts` - 报表统计 API

**当前 router.ts 只包含**:

export const appRouter = createTRPCRouter({
  auth: authRouter,

**影响**:

#### 3. 页面使用硬编码假数据 - 【严重】

**问题描述**:
所有业务页面使用的都是写死的假数据，没有调用真实 API。

**受影响页面**:
| 页面 | 假数据变量 | 应调用的 API |
|------|----------|-------------|
| `/suppliers` | `suppliersData` | `trpc.supplier.list` |
| `/projects` | `projectsData` | `trpc.project.list` |
| `/capacity` | `capacityData` | `trpc.capacity.list` |
| `/evaluations` | `evaluationsData` | `trpc.evaluation.list` |
| `/reports` | `rankingData` | `trpc.report.ranking` |

**示例代码** (`src/app/(dashboard)/suppliers/page.tsx:17-22`):
```typescript
// 临时数据
const suppliersData = [
  { id: '1', name: 'XX 科技有限公司', level: 'A', techStack: 'UE5', ... },
  { id: '2', name: 'YY 工作室', level: 'B', techStack: 'U3D', ... },
  // ...
];
```

**影响**:
系统无法显示真实业务数据，所有增删改查功能都无法实现。

#### 4. 页面路由与实际路由不匹配 - 【严重】

**问题描述**:
代码中导航链接使用的路径与实际页面路径不一致。

**代码中的路径** (`src/components/auth/login-form.tsx:26`):
```typescript
router.push('/');  // 登录后跳转到首页
```

**Dashboard 布局路径** (`src/app/(dashboard)/layout.tsx`):
所有 dashboard 页面都在 `(dashboard)` 路由组下，但导航代码中使用的是 `/dashboard/xxx`。

**Sidebar 导航** (`src/components/dashboard/sidebar.tsx:8-14`):
```typescript
const navigation = [
  { name: '仪表盘', href: '/', icon: '📊' },
  { name: '供应商管理', href: '/suppliers', icon: '👥' },
  // ...
];
```

实际页面在 `src/app/(dashboard)/suppliers/page.tsx`，访问路径是 `/suppliers`，这个是对的。但有些按钮跳转使用了 `/dashboard/suppliers/...` 导致 404。

**影响**:
部分导航跳转失败。

---

#### 5. 数据库使用 SQLite 而非 PostgreSQL - 【严重】

**问题描述**:
项目计划和 CLAUDE.md 中都明确指定使用 PostgreSQL 数据库，但实际代码使用的是 SQLite。

**证据** (`prisma/schema.prisma:9`):
```prisma
datasource db {
  provider = "sqlite"
}
```

**依赖包** (`package.json`):
```json
"@prisma/adapter-better-sqlite3": "^7.6.0",
"better-sqlite3": "^12.8.0",
```

**影响**:
- SQLite 不支持生产环境的并发访问
- 不支持 PostgreSQL 的高级特性（如 JSONB 类型）
- 部署配置中的 DATABASE_URL 需要修改

### 🟢 轻微问题（体验优化）

#### 14. 登录表单缺少 autocomplete 属性 - 【轻微】

**问题描述**:
浏览器控制台警告输入框缺少 autocomplete 属性。

**控制台警告**:
```
Input elements should have autocomplete attributes (suggested: "current-password")
```

**建议修复**:
```typescript
<Input autoComplete="username" ... />
<Input autoComplete="current-password" ... />
```

---

#### 15. 错误处理不统一 - 【轻微】

**问题描述**:
不同 API 的错误响应格式不一致。

**示例**:
- `/api/auth/login` 返回 `{ error: "..." }`
- tRPC 返回 `{ status: "ok" }` 或抛出错误

**影响**:
前端错误处理复杂化。

---

#### 16. 密码强度校验缺失 - 【轻微】

**问题描述**:
用户创建时密码没有强度校验（计划要求至少 8 位，包含大小写字母、数字、特殊字符）。

**相关代码** (`src/server/trpc/routers/user.ts:6-13`):
```typescript
const userSchema = z.object({
  password: z.string().min(6, '密码至少 6 位'),  // 只要求 6 位
  // ...
});
```

**影响**:
用户可能设置弱密码。
